> 작성자: CPO

## 1. 개요 (Overview)

**한 줄 요약**
내가 읽은 책을 Notion으로 관리하고, 웹 페이지에서 방문자에게 소개하는 개인 독서 아카이브 서비스

**배경 및 문제 정의**

- 읽은 책의 기록과 감상을 외부에 공유할 마땅한 수단이 없음
- 블로그는 관리 비용이 높고, SNS는 아카이빙에 적합하지 않음
- Notion을 CMS로 활용하면 코드 수정 없이 콘텐츠를 관리할 수 있어 운영 부담이 낮음

**목표 및 기대 효과**

- 방문자: 운영자의 독서 취향과 인사이트를 쉽게 파악
- 운영자: Notion에 책 정보 입력만으로 웹 페이지 자동 업데이트

---

## 2. 사용자 (Users)

**타겟 사용자**

- 방문자: 운영자의 지인, 채용 담당자, 독서에 관심 있는 일반 방문자
- 운영자: 서비스 소유자 본인 (Notion으로 콘텐츠 관리)

**사용자 페르소나**

| 구분   | 방문자 김민준                               | 운영자 (본인)                         |
| ------ | ------------------------------------------- | ------------------------------------- |
| 역할   | 개발자 / 이직 준비 중                       | 서비스 운영자                         |
| 목표   | 운영자가 어떤 책을 읽는지 파악하고 싶음     | 읽은 책을 기록하고 쉽게 공유하고 싶음 |
| 불편함 | 정보가 SNS에 흩어져 있어 한눈에 보기 어려움 | 블로그 포스팅은 공수가 너무 많이 듦   |

**사용자 여정**

시나리오 1 - 방문자가 책 목록을 탐색하는 경우

```
메인 페이지 진입 → 카테고리 필터 선택 → 관심 있는 책 카드 클릭 → 상세 페이지에서 한 줄 평 확인
```

시나리오 2 - 운영자가 새 책을 등록하는 경우

```
Notion DB에 책 정보 입력 → 상태를 '완독'으로 변경 → 웹 페이지 자동 반영 확인
```

## 3. 기능 요구사항 (Functional Requirements)

### 핵심 기능 목록

| 우선순위 | 기능명            | 설명                                                           |
| -------- | ----------------- | -------------------------------------------------------------- |
| Must     | 책 목록 조회      | Notion API에서 '완독' 상태의 책 목록 fetch 및 카드 형태로 표시 |
| Must     | 책 상세 정보 표시 | 제목, 저자, 표지 이미지, 별점, 한 줄 평, 읽은 날짜 표시        |
| Must     | Notion API 연동   | 데이터 fetch 및 SSG/ISR 렌더링                                 |
| Should   | 카테고리 필터     | 카테고리(Multi-select)별 책 목록 필터링                        |
| Should   | 읽은 날짜 표시    | 완독일 기준 정렬 및 표시                                       |
| Could    | 검색              | 제목, 저자명으로 클라이언트 사이드 검색                        |
| Could    | 통계 바           | 총 읽은 책 수, 올해 읽은 책 수 카운트 표시                     |

### 기능별 상세 설명

**책 목록 조회**

- Notion DB에서 `상태 = 완독` 필터 적용 후 fetch
- 기본 정렬: 읽은 날짜 내림차순
- 카드 구성: 표지 이미지, 제목, 저자, 별점

**책 상세 정보 표시**

- 상세 리뷰가 없는 경우 해당 섹션 미노출
- 표지 이미지가 없는 경우 기본 플레이스홀더 이미지 표시

**카테고리 필터**

- 전체 보기 탭 포함
- 선택된 카테고리 탭 시각적 강조

### 엣지 케이스 및 예외 처리

- Notion API 호출 실패 시: 에러 메시지 대신 빈 목록 + 안내 문구 표시
- 표지 이미지 없음: 기본 이미지(placeholder) 대체
- 책이 0권일 때: "아직 등록된 책이 없습니다" 안내 문구 표시

## 4. 비기능 요구사항 (Non-Functional Requirements)

**성능 요건**

- 초기 로딩: LCP 2.5초 이내
- ISR 또는 SSG 적용으로 Notion API 요청 최소화
- 이미지 최적화 (Next.js Image 컴포넌트 활용)

**보안 요건**

- Notion API Key는 서버 사이드에서만 사용 (클라이언트에 노출 금지)
- 환경 변수로 관리 (`.env.local`)

**접근성 요건**

- 이미지에 alt 텍스트 제공
- 키보드 네비게이션 지원

**지원 환경**

- 브라우저: Chrome, Safari, Firefox 최신 버전
- 디바이스: 모바일(375px~), 태블릿(768px~), 데스크탑(1280px~)

## 8. 의존성 및 리스크 (Dependencies & Risks)

**외부 의존성**

- Notion API: 책 데이터 소스 (`@notionhq/client` SDK 사용)
- Open Library Covers API: ISBN 기반 책 표지 이미지
- Vercel: 배포 환경

**추가 패키지**

- `@notionhq/client`: Notion 공식 SDK
- `notion-to-md` (선택): Blocks API → Markdown 변환

**주요 리스크 및 완화 방안**

| 심각도      | 리스크                                                             | 완화 방안                                                                                              |
| ----------- | ------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------ |
| 🔴 Critical | Notion 내부 이미지 URL 1시간 만료 → ISR 주기와 충돌                | 상세 리뷰에 Notion 내부 이미지 업로드 금지, 외부 URL만 허용하는 운영 정책 수립                         |
| 🟠 Major    | CSP 설정과 외부 이미지 도메인 충돌 (이미지 미표시)                 | `next.config.ts`의 CSP `img-src`에 `covers.openlibrary.org` 등 추가, `images.remotePatterns` 함께 설정 |
| 🟡 Minor    | Notion API Rate Limit (초당 3회) — 빌드 시 책 100권 이상일 때 지연 | ISR/SSG로 캐싱, 필요 시 빌드 타임 요청에 딜레이 추가                                                   |
| 🟡 Minor    | Notion DB 스키마 변경                                              | 필드명 상수화하여 변경 시 한 곳만 수정                                                                 |

**Notion Blocks API 파싱 주의사항**

- 지원 블록 타입 범위를 명시적으로 제한 (`paragraph`, `heading_1~3`, `bulleted_list_item`, `image`, `quote`, `callout`)
- 미지원 블록 타입은 렌더링 스킵 처리
- 중첩 블록(`has_children: true`) 사용 금지 운영 정책 또는 재귀 호출 로직 설계 필요
